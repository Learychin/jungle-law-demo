# 丛林法则 Demo

这是一个可直接打开的本地网页 Demo。

## 打开方式

直接用浏览器打开：

`index.html`

如果浏览器有本地文件限制，也可以在 `demo/` 目录启动本地服务：

```bash
python3 -m http.server 8765 --bind 127.0.0.1
```

然后访问：

`http://127.0.0.1:8765/`

## 双人在线 PeerJS 试玩

当前 Demo 已接入 PeerJS 在线房间：

1. 房主打开页面，点顶部“开双人链接”。
2. 房间创建成功后，点“复制邀请”，把链接发给朋友。
3. 朋友打开带 `?room=` 的链接后会自动加入。
4. 房主浏览器负责真实规则结算；朋友端只发送点击动作，收到房主同步后刷新牌桌。

这一版是朋友试玩 MVP：适合双方同时在线对战，不做账号、匹配、断线恢复和反作弊。PeerJS 浏览器脚本已放在 `peerjs.min.js`，部署到 GitHub Pages 或 EdgeOne 时会跟着静态站点一起发布。

默认使用 PeerJS 官方云信令服务。如果国内玩家能打开站点但进不了房间，通常是信令服务或 WebRTC 连接问题，可以后续自建 PeerServer，然后在页面加载前设置：

```html
<script>
  window.JUNGLE_PEER_CONFIG = {
    host: "你的-peer-server-域名",
    port: 443,
    path: "/",
    secure: true
  };
</script>
```

GitHub Pages 可以继续作为海外/临时发布；如果国内访问 GitHub Pages 慢或不稳定，把整个 `demo/` 作为静态站点部署到腾讯 EdgeOne 更合适。EdgeOne 会改善页面和素材加载，但 PeerJS 的实时连接仍然依赖 PeerServer / WebRTC 本身。

## 已实现

- 玩家一 vs AI 对手模式，默认开启
- 可切换回 1V1 热座模式
- 可用 PeerJS 创建双人在线房间，发链接给朋友同步对战
- 自动洗牌、发 5 张初始手牌
- 私有卡组抽牌
- A/B/C 公共池抽牌，A 类按试玩建议为 3 食物
- 天空/陆地/水生三空间，每方各 3 个格位
- 按生物空间属性限制上场位置
- 天空和两栖生物现在可以花 1 食物换线移动
- 每回合基础 +3 食物，水生空间每张 +1，最多 +3
- 开局资源改成：先手 6 食物，后手 2 食物
- 生物上场，最低 1 食物
- 刚上场不能攻击
- 攻击玩家
- 攻击对方生物，结算阵亡/受伤/不能攻击
- 战斗/备战/休息状态
- 休息状态花 1 食物恢复，但每回合最多复工 1 只动物
- 场上生物当作 1 食物消耗，每回合限 1 次
- 打玩家时，如果对手场上还有生物，需要额外 +1 食物
- 打玩家时，防守方现在可以派合法动物格挡，并支付该动物当前防御值
- 生物挨打时，防守方现在也可以自己掉血，把这口伤害替它扛掉
- AI 打你的生物时，现在也会走同一套“玩家承受打击 / 让动物承受伤害”结算，不再偷偷跳过这层选择
- 生命值归 0 判定胜利
- 战斗日志
- 本局自动记录，结束后保存在当前浏览器
- 可导出“本局 JSON”和“全部本地记录 JSON”
- 可操作空格和可攻击目标会高亮，行动路线会实时提示
- 推荐路线现在会用高亮提示手牌、落点、目标和按钮
- 行动按钮会写清费用、攻击/防御结算和额外消耗
- 桌面端按空格，可以执行当前高亮动作
- 中间新增“观众热度”，技能 / 击杀 / 节目卡 / 连招会把热度抬高，4 / 7 档会掉喝彩奖励
- 中间新增“本回合串烧”，会把这回合已经打出来的连段和节目效果串起来
- 推荐路线现在会直接高亮到手牌、落点、目标和推荐按钮，减少“看懂了但不知道点哪”的成本
- 场边播报现在会多一嘴主持人 / 观众碎嘴，推荐按钮的标签也更像真的在带局
- 顶部三段路线条现在会跟着当前最佳手变口吻，不再永远是同一套死文案
- 热度快到奖励线时，中间提示会明显更兴奋，直接告诉你这手能不能把奖励抖下来
- 单屏左右布局，玩家 / 行动区 / AI 同时可见，并对窄屏做了响应式压缩
- 每回合会随机翻出 1 张“节目卡”（轻量自然灾害）
- AI 现在有 4 种人格：`稳健派 / 莽夫流 / 老六流 / 戏精流`

## 已接上的技能

- 主动技能
  - `偷盗`：花 1 食物，随机偷对手 1 张手牌
  - `鹰眼`：花 2 食物，先看对手手牌，再随机偷 1 张
  - `吸引`：花 1 食物，获得一回合嘲讽
  - `毒牙`：花 1 食物，下一次打生物时，打不死也能让目标休息
  - `采摘`：花 2 食物，试玩版先简化为给自己回 1 血
  - `百兽王吼`：花 2 食物，压制对手陆地生物；本回合和对方下回合都有效
  - `谁是百兽王？`：花 2 食物，试玩版先简化成常驻“兽王状态”直到阵亡
- 被动技能
  - `美味` / `超级美味`
  - `毒腺` / `猛毒` / `携带病毒`
  - `天空攻击指挥官`
  - `海洋防御指挥官`
  - `狼群`
  - `势不可挡`：对手用动物替玩家格挡时，仍会漏 2 点伤害

当前 Demo 会把技能写进按钮文案、悬浮提示、状态徽章和日志里，方便第一次试玩时看懂。

## 已接上的节目卡

- 每位玩家在自己回合开始时，都会随机翻到 1 张节目卡
- 节目卡是轻量自然灾害系统，当前不进入手牌，直接显示在中间栏
- 节目卡费用：
  - 单点类 `-2`
  - 双点类 `-3`
  - 全灭类 `-4`
- 触发规则：
  - 需要对应空间的双方场上都至少有生物
  - 只会影响对应空间
  - 试玩版会自动选择“己方更不亏、对方更肉疼”的目标
- 当前已接入全部 9 张节目卡：
  - `干旱 / 森林大火 / 大地震`
  - `雾霾 / 龙卷风 / 雷暴`
  - `赤潮 / 海洋污染 / 海啸`

这一版的目标不是完全复刻原始指向式选目标，而是先把“有节目效果、容易点、容易懂”的节奏接起来。

## AI 对手

AI 会在玩家一结束回合后自动行动。当前 AI 策略：

- 优先寻找斩杀玩家的攻击
- 如果节目卡能明显赚交换，AI 会先放节目卡
- 更偏向先处理高价值生物，优先击杀，其次打成休息
- 选择较高价值生物上场，偏好能产生水生经济的位置
- 根据手牌和食物抽私有牌或公共 A/B/C 牌，但不再那么迷恋公共 A
- 不会再像之前那样死盯几张便宜功能牌反复刷存在感
- 食物紧张时，可能牺牲低价值或 0 攻生物换 1 食物，但也遵守每回合一次的上限
- 打玩家前会先判断要不要逼你交动物格挡，防守时也会评估该不该用动物替自己扛
- 会在合适的时候让两栖/天空生物换线，去开新攻击路线或蹭水生收益
- 可以手动切换 AI 人格，让它更偏向稳健控场、莽脸、阴招或节目效果

AI 不读取玩家手牌，只根据公开场面、自己的手牌和资源行动。

## 基本操作路线

1. 看高亮的手牌、格子、目标和按钮。
2. 可以先点“抽自己牌 / 抽强牌A / 抽中牌B / 抽小牌C”补牌。
3. 要上场：点一张手牌，再点左侧亮起来的空格。
4. 要攻击：点你场上的一只生物，再点“攻击 AI”或点右侧亮起来的目标生物。
5. 想换线：选一只天空或两栖生物，点“移动空间 -1”，再点亮起来的空格。
6. 如果对面要直接打玩家，场上亮起来的己方动物可以替你格挡；也可以点“玩家承受打击”。
7. 如果对面要打你的生物，中间会出现“玩家承受打击”按钮，可以用玩家生命保住关键动物。
8. 想把休息生物叫醒：选中它，点“复工 -1”；这一招每回合只能用 1 次。
9. 如果不想自己算费用，先看按钮上的费用说明和战斗日志。
10. 想重选时，点“取消选择”。
11. 没有想做的动作时，点右上角“结束回合”，AI 会自动行动。

页面现在采用单屏左右布局：左边是你，中间是行动和日志，右边是 AI。

## 记录保存

- 现在已经会把每一局完整记录保存在当前浏览器的 `localStorage`
- 可以随时导出本局 JSON，或者导出全部历史记录 JSON
- 如果想把记录“自动写回 GitHub 仓库”，静态网页本身还做不到；那需要额外的后端或 GitHub API 授权流程

## 调试钩子

页面暴露了一个本地调试入口：

`window.__JUNGLE_DEBUG__`

它主要给自动测试和后续平衡脚本用，可以直接读当前对局、选牌、重开和执行规则函数。

现在仓库里还附带了一个本地平衡实验台：

`balance-lab.mjs`

可以直接运行：

```bash
node balance-lab.mjs
```

它会批量跑 AI 自对局，吐出先后手胜率、平均回合数、事件频率、节目卡 / 技能 / 常见上场生物等统计，用来给下一轮平衡补丁提供依据。
